Phase 1 – Central Rules

Goal

Establish GLS as the single source of truth for labour rules.

Scope

  • Move labour rules from LP to GLS
  • Rule lifecycle management (create, edit, assign, version, audit)
  • Multi-tenant + single-tenant UI
  • CUP-based access control
  • Integration with CFM and SLM
  • Rule assignment notifications

Success Criteria

  • All rules managed in GLS
  • LP no longer used for configuration
  • Rules consumed by downstream systems
  • UI fully accessible via iQ360

Phase 2 – AI Discovery & Scoping

Goal

Enable insight generation and rule performance understanding.

Scope

  • Analyse rule usage and overrides
  • Integrate timecards and schedules
  • Scenario modelling via SLM
  • "What if" comparisons (past vs future)

Success Criteria

  • Users can compare rule scenarios
  • Insights available in iQ360
  • Clear optimisation opportunities identified

Phase 3 – AI Deployment

Goal

Enable automated optimisation and execution.

Scope

  • AI-driven recommendations
  • Rule and schedule adjustments
  • Integration with Auto Scheduling & Dynamic Labour
  • Approval-based and automated execution

Success Criteria

  • Improved labour cost %
  • Increased schedule efficiency
  • Reduced manual intervention

Architecture Principles

  • GLS = Control plane – Central management and source of truth
  • SLM = Intelligence layer – Scenario modelling and analysis
  • Execution decoupled from rules – Flexible rule application
  • Unified UI via iQ360 micro-frontends – Seamless user experience

Risks & Mitigations

HIGH
Data inconsistency across systems – Misalignment between rule definitions and execution could impact labour compliance and cost accuracy.
MEDIUM
Latency in event-driven architecture – Delays in event propagation could affect real-time rule updates and schedule adjustments.
HIGH
Ownership ambiguity – Unclear responsibility across teams could lead to delayed decisions and unresolved issues.
MEDIUM
Alert fatigue – Excessive notifications from the Decision Engine could reduce user engagement and mask critical alerts.
